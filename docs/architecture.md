# Architecture

## Overview

ClawOps is a Spring Boot backend that acts as an infrastructure control plane. It orchestrates operations on remote servers via SSH — no agents or daemons need to be installed on the managed machines.

```
 Browser / API Client
        |
        v
 ┌─────────────────────┐
 │  Spring Boot API     │
 │  (REST + WebSocket)  │
 │                      │
 │  ┌──────────────┐    │     ┌──────────────┐
 │  │ SSH Service   │────┼────>│ Remote Server │
 │  └──────────────┘    │     └──────────────┘
 │  ┌──────────────┐    │     ┌──────────────┐
 │  │ DNS Provider  │────┼────>│ Cloudflare   │
 │  │ Adapters      │    │     │ Namecheap    │
 │  └──────────────┘    │     └──────────────┘
 │                      │
 │  ┌──────────────┐    │
 │  │ PostgreSQL    │    │
 │  └──────────────┘    │
 └─────────────────────┘
```

**Core pattern:** Java backend orchestrates -> uploads script -> executes via SSH -> tracks logs/status

## Module Map

The project is organized into 11 modules, each with its own package under `com.openclaw.manager.openclawserversmanager`:

| Module | Package | Purpose | Key Classes |
|--------|---------|---------|-------------|
| **auth** | `.auth` | JWT authentication, login, refresh, logout | `AuthController`, `JwtTokenProvider`, `JwtAuthenticationFilter` |
| **users** | `.users` | DevOps user management | `UserController`, `UserService`, `User` entity |
| **servers** | `.servers` | Server inventory and connections | `ServerController`, `ServerService`, `Server` entity |
| **secrets** | `.secrets` | AES-GCM encrypted credentials | `SecretController`, `SecretService`, `EncryptionService` |
| **ssh** | `.ssh` | SSH command execution, SFTP | `SshService`, `CommandResult` |
| **terminal** | `.terminal` | WebSocket browser terminal | `TerminalWebSocketHandler`, `TerminalSessionManager` |
| **deployment** | `.deployment` | Script library and async jobs | `DeploymentScriptController`, `DeploymentJobService`, `ScriptRunner` |
| **templates** | `.templates` | Agent template provisioning | `AgentTemplateController`, `TemplateService` |
| **domains** | `.domains` | DNS automation (Cloudflare, Namecheap) | `ProviderAccountController`, `ManagedZoneController`, `DomainAssignmentController`, `SslController` |
| **audit** | `.audit` | Operation audit logging | `AuditController`, `AuditService` |
| **common** | `.common` | Shared utilities, exceptions | `GlobalExceptionHandler`, `ErrorResponse` |

### Module Communication

Modules communicate through **services only**. A module never accesses another module's repository directly.

```
Controller -> Service -> Repository (own module)
                |
                +-> Other Module's Service (cross-module)
```

## Data Flows

### Add Server Flow

```
POST /api/v1/servers
  -> ServerService.createServer()
     -> ServerRepository.save()
     -> DomainAssignmentService.autoAssignIfConfigured()
        -> DnsProviderAdapter.createRecord()
        -> SslService.provisionCertificate() [async]
           -> SshService.executeCommand() [install nginx/certbot]
           -> SshService.uploadFile() [nginx config]
           -> SshService.executeCommand() [certbot --nginx]
     -> AuditService.log(SERVER_CREATED)
```

### Run Deployment Script Flow

```
POST /api/v1/deployment-jobs
  -> DeploymentJobService.triggerJob()
     -> Check no RUNNING job on server
     -> Create DeploymentJob (status: PENDING)
     -> ScriptRunner.run() [@Async]
        -> Set status RUNNING
        -> SshService.uploadFile() [/tmp/{jobId}.sh]
        -> SshService.executeCommand() [bash /tmp/{jobId}.sh]
        -> Capture stdout/stderr as logs
        -> Set status COMPLETED or FAILED
        -> SshService.executeCommand() [rm cleanup]
     -> AuditService.log(JOB_TRIGGERED)
```

### Deploy Agent Template Flow

```
POST /api/v1/agent-templates/{id}/deploy
  -> TemplateService.deployTemplate()
     -> Load template's installScript
     -> DeploymentJobService.triggerTemplateJob()
        -> Create DeploymentJob (scriptName: "template:{name}")
        -> ScriptRunner.run() [@Async]
     -> AuditService.log(TEMPLATE_DEPLOYED)
```

### SSL Provisioning Flow

```
POST /api/v1/ssl-certificates
  -> SslService.provisionCertificate()
     -> Check/install nginx + certbot on server
     -> Upload nginx site config via SFTP (temp file + sudo mv)
     -> Remove default nginx site
     -> Enable site, test config, start nginx
     -> Wait for DNS propagation (6 attempts, 10s intervals)
     -> Run certbot --nginx
     -> Record SslCertificate in database
```

## Database Schema

All tables use UUID primary keys generated by PostgreSQL (`gen_random_uuid()`).

| Table | Purpose | Key Columns |
|-------|---------|-------------|
| `users` | DevOps users | id, email, username, password_hash, role |
| `refresh_tokens` | JWT refresh tokens | id, user_id, token, expires_at |
| `audit_logs` | Operation audit trail | id, user_id, action, entity_type, entity_id, details, ip_address |
| `secrets` | Encrypted credentials | id, name, type, encrypted_value, iv, description |
| `servers` | Server inventory | id, name, hostname, ip_address, ssh_port, ssh_username, auth_type, credential_id |
| `provider_accounts` | DNS provider accounts | id, name, provider_type, encrypted_credentials |
| `managed_zones` | DNS zones for subdomains | id, domain, provider_account_id, auto_assign |
| `domain_assignments` | Subdomain -> server mappings | id, zone_id, subdomain, record_type, target, resource_id |
| `domain_events` | DNS operation history | id, assignment_id, event_type, details |
| `ssl_certificates` | SSL cert records | id, server_id, domain, status, issued_at, expires_at |
| `deployment_scripts` | Bash script library | id, name, script_content, script_type |
| `deployment_jobs` | Script execution records | id, server_id, script_id, status, logs, error_message |
| `agent_templates` | Agent template definitions | id, name, agent_type, install_script |

Migrations are managed by Flyway and located in `src/main/resources/db/migration/`.

## Async Execution

Deployment jobs and template deployments run asynchronously using Spring's `@Async` with a dedicated thread pool:

- **Executor**: `deploymentExecutor` (core: 4, max: 10, queue: 50)
- **Bean**: `ScriptRunner` is a separate `@Service` bean (not an internal method) so the Spring proxy intercepts the `@Async` annotation
- **Concurrency control**: only one job can be RUNNING per server at a time

## Security Model

See [security.md](security.md) for full details. Summary:

- **Authentication**: JWT Bearer tokens (access + refresh)
- **Roles**: ADMIN (full access) and DEVOPS (read + execute, no user management or deletions)
- **Encryption**: AES-256-GCM for secrets at rest, with per-secret random IV
- **No public signup**: all users created by admin
